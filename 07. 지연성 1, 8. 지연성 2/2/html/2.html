# 이터러블 중심 프로그래밍에서의 지연 평가 (Lazy Evaluation) - 제때 계산법 - 느긋한 계산법 - 제너레이터/이터레이터 프로토콜을 기반으로 구현 - 가장 필요한때까지 평가를 미루다가 정말 필요할 때 해당하는 코드들을 평가하면서 값을 만들어나가는 기법 - ex) 큰 크기의 배열을 미리 만들어 놓는것이 아니라 필요한 값을 뽑을 때, 배열의 안쪽 값을 만들어 나가면서 값을 만드는 것을 최소화하면서 연산을 효율적으로 함 - 이전의 자바스크립트에서는 공식적인 구현이 어려웠지만 ES6에서의 제너레이터/이터레이터 프로토콜을 기반으로 구현 ### L.map ### L.filter ### range, map, filter, take, reduce 중첩 사용 ### L.range, L.map, L.filter, take, reduce 중첩 사용 ### map, filter 계열 함수들이 가지는 결합 법칙 - 특정한 방식으로 평가 방식을 바꿔도 똑같은 결과를 만든다. - 사용하는 데이터가 무엇이든지 - 사용하는 보조 함수가 순수 함수라면 무엇이든지 - 아래와 같이 결합한다면 둘 다 결과가 같다. [[mapping, mapping], [filtering, filtering], [mapping, mapping]] = [[mapping, filtering, mapping], [mapping, filtering, mapping]] - 지연평가 (지연해두는 방식) - 이전 자바스크립트에서는 공식적인 값이 아니라 라이브러리나, 어떠한 형태로 구현을 해야 됐었음 - 자바스크립트의 공식적인 값을 통해서 함수와 함수가 리턴값을 통해 지연되었고, 평가 시점을 JS와 개발자가 약속된 규약을 가지고 만들어 나갈 수 있게 됨 ## 결과를 만드는 함수 reduce, take - map이나 filter같은 함수는 배열이나, 이터러블한 값의 안쪽에 있는 원소들에게 함수들을 합성해주는 역할 (지연성을 가질 수 있음) - reduce나 take는 안쪽에 있는 값들을 꺼내서 더한다거나 하는 식으로 값을 유지시키는게 아니라 깨뜨리는 역할 (즉, 연산 시작을 알리는 시작점) ### reduce ### take, find
